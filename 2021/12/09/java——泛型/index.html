java——泛型

1.理解泛型

什么是泛型？为什么要使用泛型？

Java 泛型（generics）是 JDK 5 中引入的一个新特性, 泛型提供了编译时类型安全检测机制，该机制允许程序员在编译时检测到非法的类型。

泛型的本质是参数化类型，也就是说所操作的数据类型被指定为一个参数。 泛型，即“参数化类型”。就是将类型由原来的具体的类型参数化（称之为类型形参），然后在使用/调用时传入具体的类型（类型实参）。

泛型可以用在类、接口和方法中，分别被称为泛型类、泛型接口、泛型方法。

注意：
泛型的类型参数只能是类类型，不能是简单类型。 不能对确切的泛型类型使用 instanceof 操作。 

List<String> strings = new ArrayList<String>();
// 编译报错：Illegal generic type for instanceo
if (strings instanceof List<String>) {}

2. 泛型类

泛型类的声明和非泛型类的声明类似，除了在类名后面添加了类型参数声明部分。泛型类的类型参数声明部分可以包含一个或多个类型参数，参数间用逗号隔开。

一个泛型参数，也被称为一个类型变量，是用于指定一个泛型类型名称的标识符。因为他们接受一个或多个参数，这些类被称为参数化的类或参数化的类型。

通过泛型可以完成对一组类的操作对外开放相同的接口。最典型的就是各种容器类，如：List、Set、Map。

语法： 

class 类名称 <泛型标识：可以是任意标识号>{
    private 泛型标识 /*（成员变量类型）*/ var;
    .....
}
示例： 
class Generic<T> {
    //key这个成员变量的类型为T,T的类型由外部指定
    private T key;

    public Generic(T key) { // 构造方法形参类型也为T，由外部指定
        this.key = key;
    }
}

//传入的实参类型需与泛型的类型参数类型相同，即为 Integer.
Generic<Integer> genericInteger = new Generic<Integer>(123456);

3. 泛型接口

泛型类型用于接口的定义中，被称为泛型接口。

示例 

// 定义泛型接口
interface Generator<T> {
    public T next();
}
// 实现接口一
class CarGenerator implements Generator<String> {
    @Override
    public String next() {
        return "car";
    }
}
// 实现接口二
class FruitGenerator<T> implements Generator<T> {

    private T next;

	// 非泛型方法
    public FruitGenerator(T next) {
        this.next = next;
    }

    @Override
    public T next() {
        return next;
    }

    public static void main(String[] args) {
        FruitGenerator<String> fruit = new FruitGenerator<String>("Fruit");
        System.out.println(fruit.next);
        // Fruit
    }
}

4. 泛型方法

泛型方法：是在调用方法的时候指明泛型的具体类型。 >泛型类：是在实例化类的时候指明泛型的具体类型。

4.1 泛型方法的基本用法

泛型方法的定义规则： + 所有泛型方法声明在方法返回类型之前都有类型参数声明部分（由尖括号分隔） + 每一个类型参数声明部分包含一个或多个类型参数，参数间用逗号隔开。 + 类型参数能被用来声明返回值类型，并且能作为泛型方法得到的实际参数类型的占位符。

java 中泛型标记符： + E - Element (在集合中使用，因为集合中存放的是元素) + T - Type（Java 类） + K - Key（键） + V - Value（值） + N - Number（数值类型） + ？ - 表示不确定的 java 类型 

// 泛型方法 printArray
public static <E> void printArray(E[] inputArray){
    // 输出数组元素
    Arrays.stream(inputArray).forEach(System.out::println);
}

public static void main(String args[]){
    // 创建不同的数组
    Integer[] intArray = { 1, 2, 3, 4, 5 };
    Double[] doubleArray = { 1.1, 2.2, 3.3, 4.4 };
    Character[] charArray = { 'H', 'E', 'L', 'L', 'O' };

    System.out.println("整型数组元素为：");
    printArray(intArray);

    System.out.println("双精度型数组元素为：");
    printArray(doubleArray);

    System.out.println("字符型数组元素为：");
    printArray(charArray);
}
#### 4.2 泛型方法与可变参数

   static <T> void printMsg(T... args){
       Arrays.stream(args).forEach(System.out::println);
   }

printMsg("111",111,123.23,'a',123L);

4.3 静态方法与泛型

静态方法无法访问类上定义的泛型，如果静态方法操作的引用数据类型不确定时，必须将泛型定义在方法上。

class StaticGenerator<T> {
    // 编译错误：cannot be referenced from a static context
    // public static void Show(T t) {}

    public static <T> void Show(T t) {

    }
}

5. 泛型通配符

5.1 用 ? 代替具体的类型参数

此处’？’是类型实参，而不是类型形参。也就是说此处的？和 Number、String、Integer一样都是一种实际的类型，可以把？看成所有类型的父类。

public static void main(String[] args){
    List<String> name = new ArrayList<String>();
    List<Integer> age = new ArrayList<Integer>();
    List<Number> number = new ArrayList<Number>();

    name.add("zth");
    age.add(18);
    number.add(13);
    getData(name);
    getData(age);
    getData(number);
}
public static void getData(List<?> data){
    System.out.println("data: "+data.get(0));
}

5.2 泛型的上边界

通过形如 List < ? extends Number> 来定义，如此定义就是通配符泛型值接受Number及其下层子类类型。 

public static void main(String[] args) {
    List<Integer> age = new ArrayList<>();
    List<Number> number = new ArrayList<>();
    List<Double> doubles = new ArrayList<>();

    printNumber(age);
    printNumber(number);
    printNumber(doubles);
}


public static void printNumber(List<? extends Number> data) {
    data.forEach(System.out::println);
}

Ingeter 是 Number 的一个子类，但是 Generic 不能被看作为 Generic 的子类。由此可以看出：同一种泛型可以对应多个版本（因为参数类型是不确定的），不同版本的泛型类实例是不兼容的。

5.3 泛型的下边界

通过形如 List< ? super Number> 来定义，表示类型只能接受 Number 及其三层父类类型，如Objec类型的实例。

public static void main(String[] args) {
    List<Number> number = new ArrayList<>();
    List<Object> objects = new ArrayList<>();

    printNumber(number);
    printNumber(objects);
}

public static void printNumber(List< ? super Number> data) {
    data.forEach(System.out::println);
}

注：不能在上限泛型集合中添加元素，可以在下限泛型集合中添加元素 

      List< ? super Integer> list = new ArrayList<>();
      // 可以在下限泛型集合中添加元素
      list.add(111);
      //报错： add (capture< ? super java.lang.Integer>) in List cannot be applied  to (java.lang.Object)
      list.add(new Object());

// 不能在上限泛型集合中添加元素
      List< ? extends Number> data = new ArrayList<>();
      // 报错：add  (capture< ? extends java.lang.Number>) in List cannot be applied to (java.lang.Integer)
      data.add(new Integer(111));
在日常中，只要记住 “ PECS(Producer Extends Consumer Super)原则” 即可。意思为： （1）频繁往外读取内容的，适合用上界Extends。 （2）经常往里插入的，适合用下界Super。

参考：https://www.zhihu.com/question/20400700

6. 泛型的生命周期

Java的泛型信息只存在于编译期间，编译之后所有的泛型信息都会被擦除。 

List<Integer> integerList = new ArrayList<Integer>();
List<String> stringList = new ArrayList<String>();
System.out.println(integerList.getClass() == stringList.getClass());
// true
不管是 ArrayList< Integer > 还是 ArrayList< Long >，在编译时都会被编译器擦除成了 ArrayList。

Java引入泛型是为了加强参数类型的安全性，在编译期间根据泛型声明及早发现类型转化问题，减少了类型的转换。因为在 JDK1.5 之前是没有泛型的，需要考虑向下兼容，所以对于泛型只存在于编译期间。编译成字节码后，ArrayList< Integer > 或 ArrayList< Long > 都会被擦除成 ArrayList，这样就能保持和 JDK1.5 之前版本兼容。

ArrayList<String> list = new ArrayList<String>();
list.add("hello");
Class listClass = list.getClass();
Method methodList = listClass.getMethod("add", Object.class);
methodList.invoke(list, new Integer(10));
for (Object obj : list) {
    System.out.println("obj = " + obj +", class = " + obj.getClass().getName());
}
/**
 * obj = hello, class = java.lang.String
 * obj = 10, class = java.lang.Integer
 */

在运行期间获取泛型信息

虽然泛型类在编译的时候会进行泛型擦除，不过泛型信息还是会保存在泛型类的字节码对象中。

获取到泛型类型的要求： + 必须具有真实类型的存在。 + 泛型的类型是明确的（如List是明确的，List是不明确的）。 

class Generic<T> {
    // 为了方便其他类获取，将其作为公有属性
    Class<?> clazz;

    public Generic() {
        ParameterizedType pt = (ParameterizedType)
                this.getClass().getGenericSuperclass();
        // 获取泛型参数列表，如Map<K,V>，那么返回K，V的数组
        Type[] types = pt.getActualTypeArguments();
        //因为这里只有一个泛型T，所以获取第一个元素
        clazz = (Class<?>) types[0];
    }
}

class MyGeneric extends Generic<String> {
}


class Generic1 extends Generic<String> {
}
System.out.println(new Generic1().clazz);
// class java.lang.String
