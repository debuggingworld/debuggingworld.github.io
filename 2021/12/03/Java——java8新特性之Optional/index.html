java——java8新特性之Optional

1. Optional 介绍

Optional 是在 java.util 包下的一个用于代替 null 的一个工具类

仅仅是解决 NPE(NullPointerException) 问题吗？

List<Order>  getOrders(User user){
    Optional<User> u = Optional.ofNullable(user);
    if (u.isPresent()){
        return u.get().getOrders();
    }
    return Collections.emptyList();
}

List<Order>  getOrders2(User user){
    if (null != user){
        return user.getOrders();
    }
    return Collections.emptyList();
}

第一种方法虽然使用了 Optional ，但是本质上没有区别，如何优雅的使用 Optional？

Optional 错误使用姿势 1. 调用 isPresent() 方法 2. 调用 get() 方法 3. Optional 类型作为类/实例属性 4. Optional 类型作为方法参数

解释： + 使用 isPresent() 和使用 obj != null 无任何分别 + 直接调用 get() 方法虽然不抛出 NPE 了，但如果没有值是会抛出 NoSuchElementException。

Optional 中可以依赖的是除了 isPresent() 和 get() 的其他方法：

方法 描述
public Optional map(Function<? super T, ? extends U> mapper) 如果值存在，就对该值执行提供的 mapping 函数的调用
public T orElse(T other) 如果有值则将其返回，否则返回默认值
public T orElseGet(Supplier<? extends T> other) 如果有值则将其返回，否则返回一个 Supplier 接口生成的值
public void ifPresent(Consumer<? super T> consumer) 如果值存在，就通过 consumer 消费次值，否则什么也不做
public Optional filter(Predicate<? super T> predicate) 如果值存在并且满足提供的谓词，就返回包含该值的 Optional 对象
public Optional flatMap(Function<? super T, Optional> mapper) 如果值存在，就对该值执行提供的 mapping 函数调用
public T orElseThrow(Supplier<? extends X> exceptionSupplier) throws X 如果有值则将其返回，否则抛出一个由指定的 Supplier 接口生成的异常

2. Optional 构造方式

  • Optional.of(obj)：要求传入的 obj 不能为 null，否则还抛出 NullPointerException 异常
  • Optional.empty()：value 为 null 的 Optional
  • Optional.ofNullable(obj)：如果 obj 为 null 返回 Optional.empty()，否则返回 Optional.of(obj)

Optional.of(obj) 的作用 1. 当非常明确将要传给 Optional.of(obj) 的 obj 参数不可能为 null 时, 比如刚 new 出来的对象(Optional.of(new User(…))), 或者是一个非 null 常量时; 2. 当想为 obj 断言不为 null 时, 即想在万一 obj 为 null 立即报告 NullPointException 异常, 立即修改, 而不是隐藏空指针异常

3. Optional 使用

3.1 获取值

// 存在即返回，否则提供默认值
return user.orElse(UNKNOWUSER);
// 而不是 return user.isPresent() ? user.get() : null ;


// 存在即返回，否则函数生成
return user.orElseGet(this::getDefaultUser);

// 存在即返回，否则抛出异常
return user.orElseThrow(NullPointerException::new);


// 存在才执行操作
user.ifPresent(System.out::print);
// 而不是
if (user.isPresent()){
    System.out.println(user.get());
}

3.2 map 函数

// 当 user.isPresent() 为真, 获得 用户名，否则返回 null
String name = user.map(User::getName).orElse("UNKNOW");

// java8 之前
String name2 = "";
if (user.isPresent()){
    name2 = user.get().getName();
}else {
    name2 = "UNKNOW";
}

3.3 filter 函数

Optional<String> name = Optional.of("debugging");
Optional<String> longName = name.filter(i -> i.length() > 6);
System.out.println(longName.orElse("The name is less than 6 characters"));
// debugging

Optional<String> name2 = Optional.of("admin");
Optional<String> longName2 = name2.filter(i -> i.length() > 6);
System.out.println(longName2.orElse("The name is less than 6 characters"));
// The name is less than 6 characters


Optional<String> name3 = Optional.empty();
Optional<String> longName3 = name3.filter(i -> i.length() > 6);
System.out.println(longName3.orElse("The name is less than 6 characters"));
// The name is less than 6 characters
