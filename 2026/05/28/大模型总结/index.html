二、概念关系

大模型训练
│
├── Pre-training（预训练）
│       目标：学会语言
│       方法：海量无标注文本，预测下一个 token
│
└── Post-training（后训练）  ← 微调+对齐的统称
        │
        ├── SFT（监督微调）
        │       目标：学会按指令回答
        │       数据：instruction + output
        │       必做，是后续一切的基础
        │
        └── 对齐（Alignment）
                前提：必须先完成 SFT
                目标：让回答符合人类偏好（HHH）
                │
                ├── 方案 A：RLHF（基于人类反馈的强化学习）
                │       │
                │       ├── 训练奖励模型 Reward Model
                │       │     数据：人类标注的偏好排序
                │       │     输出：给回答打分的模型
                │       │
                │       └── 强化学习优化
                │             │
                │             ├── PPO（主流 RL 算法）
                │             └── 其他 RL 算法（如 A2C 等，极少用）
                │
                └── 方案 B：DPO（直接偏好优化）
                          目标：与 RLHF 相同，但跳过奖励模型
                          数据：chosen + rejected 偏好对
                          原理：偏好数据本身隐含奖励信号
                          优势：更简单、更稳定、更省显存
                          │
                          └── 变体：ORPO / SimPO / KTO 等

核心关系

  • SFT 和对齐：串联关系，SFT 在前，对齐在后（SFT 是对齐的前置条件，不属于对齐）
  • RLHF 和 DPO：并列关系，殊途同归，DPO 是 RLHF 的简化替代
  • RLHF 和 PPO：包含关系，RLHF 是框架，PPO 是其中的算法
  • DPO 和 PPO：竞争关系，做同一件事的不同方法
  • RL 和 RLHF：RL 是通用算法类别，RLHF 是 RL 在大模型对齐场景的应用框架，RLHF ≠ RL 的一种
  • Fine-tuning 和 Post-Training：Fine-tuning 狭义指 SFT，Post-Training 广义包含 SFT + 对齐 + 其他后训练技术（蒸馏、合并等）

三、SFT 与 RL

SFT 的不可替代性

  • 定格式：预训练模型已具备一定的指令跟随能力（预训练数据中包含大量问答内容），但输出格式不稳定；SFT 将"有时能"变为"稳定能"
  • 定基调：给模型稳定可靠的任务能力，RL 是在已有能力上优化偏好，不能从零培养能力
  • 保稳定：没有 SFT，模型输出格式不可控，RL 的奖励信号无法有效计算，训练难以收敛

一句话：SFT 决定了模型"能不能用"。

RL 的不可替代性

  • 超上限：SFT 只能让模型模仿训练数据的天花板，RL 能通过探索发现比训练数据更好的策略
  • 泛化性强：SFT 容易过拟合训练数据的模式，RL 通过奖励信号泛化到更广泛的场景
  • 对齐偏好：SFT 无法表达"好与更好"的区别，只有 RL 能通过偏好反馈做到

一句话：RL 决定了模型"好不好用"。

两者的关系

预训练：打地基（学会语言）
SFT：   建框架（让建筑有基本功能，能住人）
RL：    精装修（让建筑更好用、更舒适）
  • 缺了 SFT → 模型输出不稳定，RL 训练难以收敛（奖励信号不可靠）
  • 缺了 RL → 模型能用但天花板低（只会模仿，不会超越）
