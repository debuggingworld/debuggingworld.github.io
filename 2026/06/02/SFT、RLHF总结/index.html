SFT、RLHF总结

一、基础概念

Q1: SFT 和 RLHF 分别解决什么问题？为什么不能只用一个？

参考答案：

两者解决不同层面的问题，串联关系不可替代： - SFT 解决"能不能用"：让预训练模型从"有时能回答"变为"稳定按指令回答"，定格式、定基调。SFT 只能让模型模仿训练数据的天花板 - RLHF 解决"好不好用"：在 SFT 基础上优化人类偏好，能发现比训练数据更好的策略，表达"好与更好"的区别

缺了 SFT → 模型输出格式不可控，RL 训练难以收敛（奖励信号不可靠） 缺了 RL → 模型能用但天花板低（只会模仿，不会超越）

类比：SFT = 建框架（能住人），RL = 精装修（更舒适）

Q2: RLHF 的三步训练流程是什么？每一步的输入输出分别是什么？

参考答案：

Step 1: 预训练语言模型
  输入：海量无标注文本
  输出：基础 LM（如 GPT-3）

Step 2: 训练奖励模型（Reward Model）
  输入：prompt + 多个模型生成 → 人类排序 → 转为标量奖励
  输出：能对 (prompt, response) 打出标量分数的 RM

Step 3: PPO 强化学习优化
  输入：SFT 后的 LM（策略）+ 冻结的 RM + 冻结的初始 LM（KL 参照）
  输出：对齐后的 LM

关键细节：Step 2 中人类做的是排序（ranking）而非打分（scoring），因为不同人的绝对分数不可比，但相对排序更稳定。排序通过 Elo 系统转为标量奖励。

Q3: RLHF 中为什么用排序而不用直接打分来训练奖励模型？

参考答案：

  1. 标注者标准不一：不同人对"好"的绝对标准不同，有人打 8 分有人打 5 分，但都认为 A 比 B 好
  2. 噪声更小：排序是相对比较，消除了标注者之间的尺度差异
  3. 更易标注：比较两个回答的优劣比给单个回答打绝对分数更容易
  4. 与 Bradley-Terry 模型天然契合：偏好建模的数学框架就是基于"谁比谁好"的成对比较

Q4: DPO 和 RLHF 的核心区别是什么？DPO 为什么能跳过奖励模型？

参考答案：

RLHF (PPO) DPO
训练阶段 训练 RM → PPO 优化策略 直接从偏好数据优化策略
是否需要 RM 是 否
是否需要在线采样 是（PPO 是 on-policy） 否
显存需求 高（4 个模型） 低（2 个模型）
稳定性 不稳定（RL 训练难调） 稳定（分类损失）

DPO 跳过 RM 的数学原理：在 RLHF 的 KL 约束优化问题中，最优策略有闭式解，将最优策略代入奖励函数，可以将奖励函数用策略本身表示。因此偏好数据中隐含的奖励信号可以直接转化为策略优化的分类损失，无需显式训练 RM。

DPO 损失函数本质是：增大 chosen 的对数概率、减小 rejected 的对数概率，同时以参考模型为基准控制偏离程度。

Q5: DPO 相比 RLHF 有什么劣势？

参考答案：

  1. 无法在线探索：DPO 是离线方法，只能从已有偏好数据中学习；PPO 是在线方法，策略可以生成新样本并获取反馈，有发现更优策略的潜力
  2. 数据分布偏移：DPO 的偏好数据来自旧策略生成的样本，当策略更新后数据分布已偏移，可能影响效果
  3. 奖励信号不可复用：RLHF 的 RM 可以用于推理时做 best-of-n 采样等，DPO 没有显式 RM
  4. 偏好数据质量更敏感：DPO 直接从偏好数据学习，数据中的噪声和错误直接影响策略；RLHF 的 RM 训练有一定的容错

二、SFT 深入

Q6: SFT 的损失函数是什么？训练时是所有 token 都参与 loss 计算吗？

参考答案：

SFT 使用 token 级别的交叉熵损失：

\[ \mathcal{L}_{SFT}(\theta) = -\sum_{t=1}^{T} \log p_\theta(y_t \mid y_{<t}) \]

不是所有 token 都参与 loss： - Padding token 被 mask 掉（ignore_index = -100） - 实践中常用的技巧：只对 assistant 回复部分计算 loss（assistant_only_loss=True），忽略 user/system 消息部分。因为模型不需要学习"如何提问"，只需要学习"如何回答" - 同理，prompt-completion 格式默认只对 completion 部分计算 loss

Q7: SFT 数据需要多少条？数据质量重要还是数量重要？

参考答案：

LIMA 论文（"Less Is More for Alignment"）表明：数据质量远比数量重要。

  • LIMA 用仅 1000 条高质量 SFT 数据就训练出了与 GPT-4 可比的模型
  • InstructGPT 原始论文也只用了约 13k 条人工标注的 SFT 数据
  • 低质量数据量大反而有害：会教坏模型，引入噪声和错误模式

实际经验： - 几万条高质量数据足以做指令跟随的 SFT - 数据要覆盖任务多样性，但每条数据的正确性和格式规范性更重要 - 常见错误：用模型自己生成的数据做 SFT（模型坍缩），或用低质量爬取数据

Q8: SFT 时全量微调（FFT）和参数高效微调（PEFT/LoRA）怎么选？

参考答案：

FFT 全量微调 LoRA 等参数高效微调
效果上限 更高（所有参数可调） 略低（表达能力受限）
显存需求 极高（全量梯度+优化器状态） 低（只训练少量参数）
训练速度 慢 快
学习率 较小（~2e-5） 较大（~1e-4），因为只训练新参数
适用场景 数据充足、追求最佳效果 数据较少、资源有限

关键点：LoRA 不会"丢失"预训练知识，因为原始参数冻结，只学习增量。这是做 SFT 时的安全选择。但大量数据+充分训练时 FFT 效果天花板更高。

Q9: SFT 常见过拟合问题，怎么判断和怎么解决？

参考答案：

判断：训练 loss 下降但验证 loss 上升；模型只会在训练数据中见过的指令格式上回答，换种问法就不行。

解决方案： 1. 数据侧：增加数据多样性（指令表述多样化）；数据增强（改写指令） 2. 训练侧：降低学习率；减少 epoch（SFT 通常 2-3 个 epoch 足够）；早停（early stopping） 3. 正则化：权重衰减（weight decay）；dropout 4. Loss mask：只对 assistant 回复计算 loss，不要让模型背 user 的提问 5. Packing：多条数据拼接到同一序列，提高训练效率和泛化

三、奖励模型

Q10: 奖励模型的规模应该多大？和策略模型的关系是什么？

参考答案：

业界实践表明：奖励模型需要与策略模型相当的容量。

组织 策略模型 奖励模型
OpenAI 175B 6B
Anthropic 10B-52B 10B-52B
DeepMind 70B 70B

直觉：偏好模型需要"理解"文本才能判断好坏，理解能力需要足够的模型容量。如果 RM 太小，可能无法捕捉人类偏好的细微差异，甚至被策略模型"欺骗"（reward hacking）。

Q11: 什么是 Reward Hacking（奖励作弊）？如何缓解？

参考答案：

定义：策略模型找到了奖励模型的高分漏洞，生成了 RM 给高分但实际质量很差的输出。

典型表现： - 模型输出看似格式完美但内容空洞 - 生成冗长但无实质信息的回答（RM 偏好长文本） - 输出 RM 训练数据中高分回答的模板化变体

缓解方法： 1. KL 散度惩罚：r = r_θ - λ·r_KL，惩罚策略偏离初始模型太远。没有这个惩罚，优化会生成 RM 给高分但实际是乱码的输出 2. 迭代 RLHF（Iterated Online RLHF）：随着策略更新，继续收集新输出的人类偏好标注，更新 RM 3. 集成多个 RM：降低单一 RM 的过拟合风险 4. RM 过优化检测：监控 RM 分数和真实人类评估的差距 5. DPO 替代：DPO 不依赖显式 RM，从原理上避免了 RM 被利用的问题

Q12: RLHF 中 KL 散度惩罚项的作用和原理是什么？

参考答案：

奖励函数：r = r_θ - λ·r_KL

其中 r_KL 是当前策略 π_θ 与初始模型 π_ref 之间的 KL 散度。

作用： 1. 防止策略崩溃：没有 KL 惩罚，PPO 会找到 RM 的漏洞，生成高分但无意义的文本（reward hacking） 2. 保持语言能力：约束策略不会偏离预训练模型太远，保证输出的连贯性和多样性 3. 训练稳定性：KL 约束形成信任区域（trust region），使 PPO 的参数更新更稳定

为什么不用固定的 KL 系数 λ？ 实践中 λ 需要动态调整。如果 KL 散度太大（策略偏离太远），增大 λ 加强约束；如果 KL 散度太小（策略更新太保守），减小 λ 鼓励探索。InstructGPT 使用了自适应的 KL 惩罚系数。

四、PPO 训练

Q13: RLHF 用 PPO 时，需要哪几个模型？分别的作用是什么？

参考答案：

PPO 训练同时需要 4 个模型：

  1. Actor（策略模型）：要优化的 LM，根据 prompt 生成文本。唯一被训练更新的模型
  2. Critic（价值模型）：估计状态价值函数 V(s)，用于计算优势函数（advantage），减少方差。通常和 Actor 共享部分参数或独立训练
  3. Reward Model（奖励模型）：对 (prompt, response) 打分。冻结，不更新
  4. Reference Model（参考模型）：SFT 后的初始 LM，用于计算 KL 散度惩罚。冻结，不更新

这就是 RLHF 显存需求大的核心原因：4 个大模型同时在显存中。

Q14: PPO 和 DPO 在显存需求上的差异是什么？为什么？

参考答案：

PPO DPO
需要的模型数 4（Actor + Critic + RM + Ref） 2（Policy + Ref）
是否需要在线采样 是 否
显存 约为推理的 4x+ 约为推理的 2x
训练速度 慢（需要生成→评分→更新的循环） 快（单次前向传播）

DPO 省显存的核心原因： 1. 不需要 RM（RM 的训练和推理都省了） 2. 不需要 Critic（没有 RL 的价值函数估计） 3. 不需要在线采样（不需要策略生成样本再评分，直接用离线偏好数据） 4. 损失函数是简单的分类损失，训练更稳定

Q15: PPO 训练 RLHF 时常见的训练不稳定问题有哪些？怎么解决？

参考答案：

常见问题： 1. 奖励分数持续上升但输出质量下降：Reward hacking，需要加大 KL 惩罚 2. Critic 预测不准导致优势估计偏差：Critic 需要足够的预训练或与 Actor 共享底层 3. 策略更新步长过大：PPO 的 clip 范围需要调小，确保在信任区域内更新 4. 生成退化：模型输出变得单一重复，可能是 KL 惩罚过大或奖励信号过于稀疏 5. 训练 loss 震荡：学习率过大或 batch size 过小

实践建议： - KL 惩罚系数从 0.2 开始，根据 KL 散度动态调整 - PPO clip ε 通常设为 0.2 - 使用较大的 batch size（如 512+） - Critic 和 Actor 可以共享除最后一层外的所有参数，减少显存 - 监控关键指标：RM 分数、KL 散度、输出长度分布

五、DPO 及其变体

Q16: DPO 的损失函数怎么理解？直觉是什么？

参考答案：

DPO 损失函数（简化表述）：

\[ \mathcal{L}_{DPO} = -\log \sigma\left(\beta \left[\log\frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \log\frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}\right]\right) \]

其中 y_w 是 chosen（好的），y_l 是 rejected（差的）。

直觉理解： - 第一项 log(π_θ/π_ref)(y_w)：增大 chosen 相对参考模型的概率 - 第二项 log(π_θ/π_ref)(y_l)：减小 rejected 相对参考模型的概率 - β 控制偏离参考模型的程度（β 越大越保守） - σ 是 sigmoid，将差值映射到 [0,1]

本质：在参考模型的基础上，拉大 chosen 和 rejected 之间的概率差距。参考模型起到了 RLHF 中 KL 惩罚的作用——防止策略偏离太远。

Q17: DPO 的 β 参数有什么作用？太大太小分别会怎样？

参考答案：

β 控制 DPO 优化对偏好数据的敏感程度：

  • β 太大：策略更新太保守，几乎不偏离参考模型，偏好数据的学习信号被压制，模型学不到人类偏好
  • β 太小：策略更新太激进，可能过拟合偏好数据中的噪声，输出质量不稳定，甚至偏离参考模型太远导致退化

实践中 β 通常在 0.1-0.5 之间调参。与 RLHF 中 KL 惩罚系数 λ 的作用类似。

Q18: ORPO、SimPO、KTO 这些 DPO 变体分别解决什么问题？

参考答案：

方法 解决的问题 核心思路
ORPO SFT 和对齐需要分两步做 将 SFT 的 NLL 损失和偏好优化合并为一个损失函数，一步完成 SFT+对齐。用优势比（odds ratio）衡量偏好差异
SimPO DPO 依赖参考模型 π_ref，多占一份显存 去掉参考模型，用序列自身的平均对数概率代替 KL 惩罚项。更简单、更省显存
KTO DPO 需要成对的 chosen+rejected 数据，成对标注成本高 只需要"好/坏"的二值标签（不需要成对比较），基于前景理论（prospect theory）建模。数据获取更容易

选择建议： - 数据是成对偏好 → DPO - 数据只有好/坏标签 → KTO - 想一步完成 SFT+对齐 → ORPO - 想极致省显存 → SimPO

六、实战与综合

Q19: 从零做一套 RLHF 对齐，完整的数据流程是什么？

参考答案：

Step 1: SFT 数据准备
  ├── 收集指令数据（人工编写或高质量筛选）
  ├── 格式：instruction + output 对
  └── 规模：几千到几万条高质量数据

Step 2: SFT 训练
  ├── 基于 pretrained LM 做 SFT
  ├── 只对 output 部分计算 loss
  └── 得到 SFT 模型（后续所有步骤的基础）

Step 3: 偏好数据收集
  ├── 用 SFT 模型对 prompt 生成多个回答
  ├── 人类标注者对回答排序
  └── 格式：prompt + chosen + rejected

Step 4: 训练奖励模型
  ├── 用偏好数据训练 RM
  ├── 损失函数：Bradley-Terry 偏好损失
  └── 输入 (prompt, response) → 输出标量分数

Step 5: PPO 训练
  ├── 初始化：Actor = Critic = SFT 模型，Ref 冻结
  ├── Actor 生成 → RM 评分 → 计算 KL 惩罚 → PPO 更新
  └── 监控：RM 分数、KL 散度、输出质量

（或 Step 4-5 替换为：直接用偏好数据做 DPO 训练）

Q20: 什么时候该用 RLHF/PPO，什么时候该用 DPO？

参考答案：

场景 推荐 原因
资源有限、快速迭代 DPO 简单稳定，省显存
追求最佳对齐效果 PPO 在线探索可能发现更优策略
需要 RM 做推理时筛选 PPO DPO 没有显式 RM
偏好数据是成对的 DPO 天然适配
偏好数据只有好/坏标签 KTO DPO 需要成对数据
需要持续迭代优化 PPO Iterated Online RLHF 更自然
团队缺乏 RL 工程经验 DPO PPO 调参门槛高

业界趋势：大多数团队从 DPO 起步，效果够用就不再上 PPO。Meta 的 Llama 2 用了 PPO，但 Llama 3 转向了 DPO。OpenAI 和 Anthropic 的生产模型仍主要使用 PPO（因为有足够的工程投入和在线数据流）。

Q21: Iterated Online RLHF 是什么？为什么需要它？

参考答案：

问题：标准 RLHF 中 RM 是用旧策略生成的数据训练的。当策略更新后，RM 的评分可能不再准确——新策略的输出分布与 RM 训练时见过的分布不同（分布偏移）。

解决方案：Iterated Online RLHF 1. 用当前策略生成新输出 2. 让人类对新输出做偏好标注 3. 用新数据更新 RM 4. 用更新后的 RM 继续做 PPO 5. 重复 1-4

Anthropic 在论文中明确提出这个概念。这本质上是一个 RM 和策略共同进化的过程，但带来了复杂的动态——策略和 RM 互相影响，是一个开放的研究问题。

Q22: RLHF 中的"Alignment Tax"（对齐税）是什么？怎么缓解？

参考答案：

定义：对齐训练（RLHF/DPO）在让模型更安全、更符合人类偏好的同时，可能导致模型在某些能力上退化的现象。比如： - 数学推理能力下降 - 代码生成能力下降 - 创造性下降，输出过于保守（过度对齐 / over-alignment）

原因： - RM 可能把"安全但平庸"的回答打了高分 - KL 惩罚约束了策略，限制了探索空间 - 偏好数据中"安全"维度权重过大

缓解方法： 1. 在 PPO 的奖励函数中混合预训练梯度（InstructGPT 的做法） 2. 对齐数据中加入高质量的能力数据（如数学、代码），防止这些能力退化 3. 分阶段对齐：先做 helpful 对齐，再做 safety 对齐，分别优化 4. 监控对齐过程中的能力 benchmark，发现退化及时调整

Q23: 如果让你从零设计一个 LLM 对齐方案，你会怎么选？

参考答案（开放题，考察综合判断）：

推荐路径：

  1. SFT 阶段：高质量指令数据（1-5k 条）+ LoRA 微调，只对 assistant 部分计算 loss
  2. 偏好对齐：先 DPO，效果不够再上 PPO
  3. 数据策略：
    • 优先用 AI 辅助生成偏好对（如 GPT-4 标注 chosen vs rejected），降低标注成本
    • 关键场景人工审核
  4. 评估：
    • 能力 benchmark 防止 alignment tax
    • 人类偏好评估（side-by-side 比较）
    • 安全性 red teaming

理由：DPO 作为起点性价比最高——工程简单、训练稳定、效果通常够用。只在 DPO 天花板不够（如需要在线探索更好的策略）或需要 RM 做推理时筛选时才上 PPO。

七、高频短答

Q24: SFT 用什么学习率？LoRA 和全量微调有区别吗？

  • 全量微调：~2e-5
  • LoRA：~1e-4（只训练少量新参数，需要更大的学习率才能有效更新）

Q25: RLHF 中 PPO 的 on-policy 是什么意思？

PPO 是 on-policy 算法，参数更新只用当前策略生成的数据。每生成一批数据后立即更新参数，旧数据不能复用。这导致 PPO 训练需要大量在线采样，成本高。

Q26: DPO 为什么训练更稳定？

  1. 损失函数是分类损失（类似二元分类），没有 RL 的 credit assignment 问题
  2. 不需要在线采样，不需要 Critic 估计价值函数
  3. 没有策略-价值函数的联合优化，不存在两者不匹配的问题
  4. 参考模型固定，梯度信号更明确

Q27: SFT 的 packing 是什么？为什么要做？

将多条短样本拼接到同一个序列中，提高 GPU 利用率。不做 packing 时，短样本会浪费大量 padding token。Packing 需要配合 loss mask 确保不同样本之间不互相干扰。

Q28: 为什么偏好数据标注成本高？有没有替代方案？

  • 人类排序成本高：每个 prompt 需要生成多个回答并人工比较，约需要 50k 标注样本
  • 替代方案：
    1. AI 辅助标注：用更强模型（如 GPT-4）做偏好判断（RLAIF）
    2. 宪法 AI（Constitutional AI）：Anthropic 提出的方法，用一组原则自动生成偏好数据
    3. 合成偏好数据：用规则或模型自动生成 chosen/rejected 对

Q29: RLHF 和 RLAIF 的区别？

  • RLHF：人类标注偏好数据 → 训练 RM → PPO
  • RLAIF：AI 模型（如 GPT-4）标注偏好数据 → 训练 RM → PPO

RLAIF 大幅降低标注成本，但引入了 AI 偏好的偏差。Anthropic 的 Constitutional AI 是 RLAIF 的一个具体实现。

Q30: Chat Template 在 SFT 中为什么重要？

Chat template 定义了对话的结构（如 <|im_start|>user\n...<|im_end|>），不同模型模板不同。SFT 时： - 必须使用正确的 template，否则模型学到的格式和推理时不一致 - template 中的特殊 token 需要添加到 tokenizer 中 - 推理时也必须用相同的 template
