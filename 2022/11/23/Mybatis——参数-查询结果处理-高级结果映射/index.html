Mybatis——参数\查询结果处理&高级结果映射

1. 参数处理

1. 1 参数取值方式

sql 语句有两种取值方式，分别是 #{} 和 ${}。

  • #{} 是预编译处理，${} 是字符串替换。
  • Mybatis 在处理 #{} 时，会将 sql 中的 #{} 替换为 ? 号，调用 PreparedStatement 的 set 方法来赋值；
  • Mybatis 在处理 ${} 替换成变量的值。
  • 使用 #{} 可以有效的防止 SQL 注入，提高系统安全性。

1.2 参数传递

1. 单个参数
  • mybatis 不会做任何特殊要求
  • 获取方式： #:{输入任何字符获取参数}
  • 也可用 @Param() 指定参数名

##### 2. 多个参数

  • mybatis 会进行封装，会将传进来的参数封装成 map，一个值对应两个 map。
  • id ===> { key:arg0, value:id 的值 }，{key:param1, value:id 的值}
  • userName===> {key:arg1, value:id 的值}, {key:param2 ,value:id 的值}
  • 获取方式
    • id ===> #{arg0} 或者 #{param1}
    • username ===> #{arg1} 或者 #{param2}
  • 设置参数的别名：User selectUser(@Param("id") Integer id, @Param("username") String userName)
  • 当使用了 @Param:
    • id ===> #{id} 或者 #{param1}
    • username ===> #{username} 或者 #{param2}
3. javaBean 参数

单个参数:

  • 获取方式：可以直接使用属性名
    • user.id ===> #{id}
    • user.name ===> #{name}

多个参数:

  • num ===> #{param1} 或者 @Param()
  • user===> 必须加上对象别名
    • user.id ===> #{param2.id} 或者 @Param("user") User user ===> #{user.id}
    • user.name ===> #{param2.name} 或者 @Param("user") User user ===> #{emp.username}
4. 集合或数组参数

如果是 list，MyBatis 会自动封装为 map ： { key："list"；value：userNames}

  • 没用 @Param 获取：userNames.get(0) ===> #{ list[0] } / #{ arg0[0] }
  • 有 @Param("userNames") 获取：userNames.get(0) ===> #{ userNames[0] } / #{ param1[0] }

如果是数组，MyBatis 会自动封装为 map：{ key ："array" ；value：userNames }

  • 没用 @Param 获取：userNames.get(0) ===> #{ array[0] } / #{ arg0[0] }
  • 有 @Param("userNames") 获取：userNames.get(0) ===> #{ userNames[0] } / #{ param1[0] }

2. 集合返回结果处理

1、当返回值的结果是集合的时候，返回值的类型依然写的是集合中具体的类型。

2、在查询的时候可以设置返回值的类型为 map，mybatis 会把列的名作为 key，列的值作为 value，转换到 map 中。

3、使用 @MapKey 将集合转为 map。

3. 自定义结果集 ­­­resultMap

resultType 和 resultMap 自定义结果集只能使用一个。

  • id：唯一标识， 需要和 select 标签上的 resultMap 属性进行对应。
  • type：需要映射的 pojo 对象， 可以设置别名
  • autoMapping：自动映射（默认=true），只要字段名和属性名遵循映射规则就可以自动映射，但是不建议，哪怕属性名和字段名一一对应也要显式配置映射
  • extends 如果多个 resultMap 有重复映射，可以声明父 resultMap，将公共的映射提取出来， 减少子 resultMap 的映射冗余
